## 3. Things needing input before we can action

### 3.1 GTM migration to Customer Events (P0)

Google Tag Manager (`GTM-5G4N4PSK`) is currently **hardcoded in the theme's `<head>`**. Shopify's current best practice (post-2023) is to move all analytics/pixels to **Customer Events** in admin (Settings → Customer events → Add custom pixel).

**Why it matters:**
- Hardcoded GTM fires *before* Shopify's cookie consent banner → GDPR/CCPA exposure
- Risk of double-firing if any tags are also installed via Shopify apps
- Doesn't survive theme swaps cleanly

**Questions for you:**
1. Is GTM container `GTM-5G4N4PSK` actively used? What tags does it fire (GA4, Meta Pixel, floodlights, etc.)?
2. Are any pixels *also* installed via Shopify apps or admin? (risk of duplicates)
3. Happy for us to recreate the GTM load inside Customer Events? Or do you have a stronger opinion on keeping it theme-side?

### 3.2 Lipscore API secret exposed in public JS

Not strictly SEO but you'll see it if you view source on any product page. The Lipscore `SECRET` and `API_KEY` are both baked into the frontend JS in `sections/product-template.liquid`. If that secret is write-capable, it's a risk.

Currently dependent on Lipscore support clarifying: **is the `SECRET` header read-only or read-write?** If read-only (just a public read token), we can leave it. If read-write, we need a server-side proxy.

### 3.3 Social share image upload

A new theme setting is live: **Admin → Customize → Theme settings → SEO → Default social share image**. Needs a 1200×630px brand image uploaded. Used as:
- Fallback `og:image` for any page without a product/article image
- `logo` field in homepage Organization JSON-LD (Google knowledge panel)

## 4. Known gaps / future improvements

### 4.1 Product reviews not in server-side structured data

Lipscore star ratings and review counts are fetched client-side. For server-side injection (more reliable rich snippets), we'd need either:
- Lipscore's metafield sync (if available) → we can inject `aggregateRating` directly into Product JSON-LD
- A scheduled job to pull rating data → Shopify metafield → template reads metafield

**Worth doing?** Yes — star ratings in SERP typically +15-30% CTR on product listings. Let me know if Lipscore offers the sync and we'll wire it up.

### 4.2 Collection pages — no visible breadcrumb trail

We added BreadcrumbList JSON-LD (crawlers see it) but no visible breadcrumbs on collection pages. The source site we scraped from also doesn't have them, so we matched the design. If you want visible breadcrumbs added for UX/SEO consistency, that's a design decision — not a code blocker.

### 4.3 Password overlay is cosmetic

If the site is running with the pre-launch password overlay enabled, note it's **client-side JS only**. Google can still index the HTML underneath. When you go live, make sure the overlay is disabled — or if the site needs to stay pre-launch, switch to Shopify's native storefront password (admin → Online Store → Preferences) which returns a proper HTTP 401.

### 4.4 Hreflang / internationalisation

Not in scope — Trendy Golf UK is a single-region store. If a US store launches on a separate domain later, we'll need `<link rel="alternate" hreflang="…">` tags and a canonical strategy.

  - Paste any article URL → expect `Article` result

- View source on any magazine article → confirm `<h1>` wraps the article title
- View source on any product page → confirm only one `<h1>` (the product title), no `<h4>` above it
- Run Lighthouse SEO audit on homepage, product, collection, article → expect 95-100

### 5.4 Indexing + crawl

- Once live, submit `https://trendygolf.co.uk/sitemap.xml` to Search Console
- Verify domain in Search Console + Bing Webmaster
- Check crawl stats weekly for first month — expect steady discovery
